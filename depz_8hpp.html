Detailed Description

The fifth C-ABI binding: an idiomatic C++ RAII wrapper over depz_c_api.h.

Java (Panama FFM), Node (N-API) and Python (ctypes) each wrap the SAME flat C header; this does it for C++. That the C ABI is implemented in C++ underneath is irrelevant — this layer consumes only the opaque handles and free functions of depz_c_api.h, exactly like the others. So a program written against depz::Pipeline is byte-for-byte isomorphic to the Java DepzPipeline / Node Depz programs (the same 70 C functions, one functor).

Header-only. No OpenCV. No exceptions cross the ABI: every nonzero depz_status becomes a thrown depz::Exception carrying depz_last_error(). RAII: every handle is owned by exactly one object and freed in its destructor, dependencies after the pipeline (the order the C ABI requires).

Naming maps 1:1 to the Java wrapper, with the C++ idiom of a depz:: namespace instead of a Depz prefix:

Java C++
DepzPipeline.camera(...) depz::Pipeline::camera(...)
frame.sampleDepth(x, y) frame.sampleDepth(x, y)
