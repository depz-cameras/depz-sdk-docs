Detailed Description

ONNX Runtime backend powered by the C++ ``_core`` extension.

``ep`` is the short EP slug ('cpu', 'cuda', 'openvino', ...); the
wrapper resolves it to ORT's full ``XxxExecutionProvider`` name and
refuses an EP not compiled into this ORT build (no silent CPU fallback).
No ``import onnxruntime`` required — all ORT calls go through C++.
