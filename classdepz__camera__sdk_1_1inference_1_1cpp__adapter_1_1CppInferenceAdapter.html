Detailed Description

Wraps a ``_core.inference.*`` C++ object for ``DepthPipeline``.

Hot-path discipline:
- ``submit(canvas_l, canvas_r)`` → C++ ``submit_sync`` (GIL released
  for entire preprocess + engine.compute block) → pre-completed future.
- ``postprocess(raw, canvas_layout)`` → C++ ``postprocess`` (squeeze
  + crop only; nanoseconds).

Everything between grabbing a canvas and producing the 2D output runs
in C++ with no Python interleaving.

Push canvas-space intrinsics to the inference. Default
no-op — most backends don't need them (classical matchers,
relative-depth nets, stereo nets that already consume a
rectified pair).

Called by the pipeline from
:meth:`DepthPipeline._rebuild_calibration_geometry` whenever
the canvas remap is rebuilt (start, calibration update, mode
change). Focus-aware monocular metric-depth architectures
(UniDepth, Metric3D, DepthPro) override to remember these
and use them in their preprocess to produce metric depth.

Non-blocking submit: runs C++ submit_sync on the worker thread.

submit_sync releases the GIL for the entire preprocess+compute block,
so the pipeline thread can grab/rectify the next frame in parallel.
Inputs are copied here because the pipeline reuses pre-allocated frame
buffers — without a copy, the worker could read the next frame's data.
