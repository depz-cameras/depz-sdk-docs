Detailed Description

Factory + enumeration of available inference engines.

Engines are identified by a **slug string** like `"openvino-CPU"` or
`"onnx-cuda"` — `f"{kind}-{device or ep}"`. There's no descriptor
class; the slug is the identity.

Public API:
    available_engines()         → list[str], **sorted by priority**
    recommend_engine()           → str | None — best one for this host
    make_engine(slug, model)     → construct + return InferenceEngine
    parse_engine_slug(slug)      → (kind, device_or_ep) — small helper
                                   for UIs that want the parts split.

◆ available_engines()

list[str] depz_camera_sdk.engines.registry.available_engines ( ) 
Probe the host: enumerate every loadable backend × device/EP.

Returned list is **sorted by priority** — `available_engines()[0]` is
the recommended choice (also returned by `recommend_engine()`).
Raises :class:`KeyError` if a discovered combination isn't in
:data:`_PRIORITY_ORDER` — that means a new device / EP showed up
that the SDK's priority table doesn't account for; add it there.

str  model_path,
**  kwargs

Construct the engine identified by `slug` against `model_path`.
`slug` is one of the strings returned by :func:`available_engines`,
e.g. ``"openvino-CPU"`` or ``"onnx-cuda"``. User input is accepted
in any case — both kind and device/EP tail are normalized to the
canonical form each backend expects (OpenVINO devices uppercase,
ONNX EPs lowercase).

Extra kwargs are passed through to the backend constructor.
The most useful one is ``cache_dir=`` — backends that cache
compiled models on disk (currently OpenVINO) will use
``<cache_dir>/<backend>/`` as the cache root. Defaults to
:func:`depz_camera_sdk.get_cache_dir` when omitted.

◆ parse_engine_slug()

tuple[str, str] depz_camera_sdk.engines.registry.parse_engine_slug ( str  slug) 
Split an engine slug into (kind, device_or_ep). For
``"openvino-CPU"`` returns ``("openvino", "CPU")``; for
``"onnx-cuda"`` returns ``("onnx", "cuda")``. The kind is
lowercased — user input is accepted in any case
(``"OPENVINO-CPU"``, ``"Openvino-cpu"`` all parse to
``("openvino", ...)``); the tail is returned verbatim and
normalized later by :func:`make_engine` per-kind. Raises
:class:`ValueError` if the slug doesn't have the expected
`kind-tail` shape.

◆ recommend_engine()

Optional[str] depz_camera_sdk.engines.registry.recommend_engine ( ) 
Best available engine slug for this host, or None if none loadable.

Override via the ``DEPZ_ENGINE`` env var (e.g.
``DEPZ_ENGINE=openvino-CPU``). Without override, low-power Intel
iGPUs are deprioritized below CPU (we benchmarked LightStereo on a
UHD Graphics box: CPU did 33 fps, iGPU 14 fps — picking the iGPU
by virtue of "G > C" is a footgun).
