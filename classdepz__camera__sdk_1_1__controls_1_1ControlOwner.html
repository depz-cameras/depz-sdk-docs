Detailed Description

Mixin providing ``get`` / ``set`` / ``to_dict`` and the shared
coerce-clamp-snap logic on top of a :meth:`_controls` hook the host
class supplies.

Hosts override :meth:`_controls` to return their descriptor list. The
rest is inherited:
  - :meth:`control_get(name)` → current attribute value.
  - :meth:`control_set(name, value)` → look up descriptor, coerce,
    store. Subclasses that need side effects (matcher rebuilds,
    etc.) override and call ``super().control_set(...)`` first.
  - :meth:`controls_to_dict()` → snapshot every current value (UI
    live state).

``control_get / control_set / controls_to_dict`` are the canonical
accessors at every layer and in every language binding.

Return the current value of one control. Reads ``self._<name>`` —
every control owner backs its controls with private storage and
exposes a read-only ``@property`` of the same bare name (so
direct ``host.<name> = x`` raises ``AttributeError`` instead of
silently bypassing coerce / side effects / worker synchronization).

Apply one control update. Default: look up descriptor, coerce
+ clamp by ``kind``/``min``/``max``/``step``, ``setattr`` onto
``self._<name>``. Returns the value actually stored.

Subclasses override to add side effects (e.g. SGBM rebuilds
its matcher when controls change) — call ``super().control_set(...)``
for the coerce/store, then act on the new value.

Reimplemented in depz_camera_sdk.pipeline.depth_pipeline.DepthPipeline, depz_camera_sdk.calibration.opencv_basic.OpenCVBasicCalibration, and depz_camera_sdk.calibration.base.Calibration.
