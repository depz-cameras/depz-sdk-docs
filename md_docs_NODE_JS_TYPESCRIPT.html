iStereoLab SDK  0.0.10
C++17 stereo vision library with Python bindings
JavaScript and TypeScript Binding

The Node SDK is published from bindings/node as @depz/istereolab. It is usable from plain JavaScript and from TypeScript.

Runtime path:

JavaScript / TypeScript
-> @depz/istereolab
-> native Node .node addon
-> flat C ABI
-> C++ core

The JavaScript API and the TypeScript API are the same runtime API. TypeScript gets declarations from bindings/node/dist/*.d.ts after npm run build.

Requirements

  • Linux x86_64
  • Node.js 18 or newer
  • DEPZ native runtime libraries bundled in the npm tarball, or available from a local SDK build
  • A stereo camera such as /dev/video0, or an MKV replay file for pipeline examples that need frames

End User Quickstart

Install a prebuilt local tarball:

npm init -y
npm install /path/to/depz-istereolab-node-<version>-linux-x86_64.tgz

Check that the native addon loads:

node --input-type=module -e \
'import { loadNativeAddon } from "@depz/istereolab"; const v = loadNativeAddon().version(); console.log(`depz C ABI ${v.major}.${v.minor}`)'

Run an example from this repository:

node examples/node/js_version_probe.mjs
node examples/node/bench_rectify_camera.mjs \
resources/config/camera/calibration_stereo.json \
/dev/video0

Build From Source

From the repository root:

export ISTEREOLAB_LIBS_DIR=$HOME/libs
./sdk_master.sh build
cd bindings/node
npm install
npm run native:build
npm run build
npm test

If DEPZ_LIBRARY is not set, the package looks for build/libdepz_istereolab.so from the repository root.

Build the self-contained npm tarball:

export ISTEREOLAB_LIBS_DIR=$HOME/libs

JavaScript Use

Use ESM imports in .mjs or in a package with "type": "module".

import { writeFileSync } from "node:fs";
import { DepzPipeline, Stream } from "@depz/istereolab";
const pipe = DepzPipeline.build({
device: "/dev/video0",
calibrationFile: "resources/config/camera/calibration_stereo.json",
model: "resources/models/lightstereo_s_sceneflow_general_opt_256_512.onnx",
engine: "openvino-CPU",
rectified: true,
disparity: true,
depth: true,
});
try {
pipe.start();
const frame = pipe.get(5.0);
if (!frame) throw new Error("no frame within timeout");
try {
const jpeg = frame.encodeJpeg({ stream: Stream.DISPARITY, quality: 90 });
writeFileSync("preview_disparity.jpg", jpeg);
} finally {
frame.close();
}
} finally {
pipe.close();
}

The package exports the same classes plus TypeScript interfaces for options, frames, image copies, metrics, and native handles.

import {

Standalone Classes

DepzPipeline.build({...}) is the all-in-one path: it owns the source, the calibration, and the inference, and runs decode + split + rectify + inference on its worker thread. When you need finer control — your own capture loop, a custom inference cadence, or rectification without inference — drive the building blocks directly. Each is created through a static factory and freed with close().

Class Factories Key methods
DepzSource .camera(opts?), .mkv(path), .pipeline(gstPipeline) open(), grab(timeoutS?, cap?), close(), closeDevice()
DepzCalibration .fromFile(path), .fromJson(json), .fromWeb(url) info(), setCanvasFitMode(mode), prepare(w, h), canvasLayout(), rectify({left, right, ...})
DepzInference .create({arch, model, engine}) setIntrinsics(fx, fy, cx, cy), disparity({left, right, ...})
DepzRecorder .create({path, width, height, fps, fourcc?}) write(buffer, ptsUs), close()

Rectify a decoded stereo pair and run direct disparity, no pipeline:

import { DepzCalibration, DepzInference } from "@depz/istereolab";
// left / right are already-decoded BGR image Buffers (e.g. from your own
// capture + JPEG decode + split-at-mid-column step).
const calib = DepzCalibration.fromFile(
"resources/config/camera/calibration_stereo.json",
);
const inf = DepzInference.create({
arch: "lightstereo",
model: "resources/models/lightstereo_s_sceneflow_general_opt_256_512.onnx",
engine: "openvino-CPU",
});
try {
calib.prepare(width, height);
const rect = calib.rectify({ left, right, width, height });
const disp = inf.disparity({
left: rect.left,
right: rect.right,
width: rect.width,
height: rect.height,
});
// disp: { data: Buffer (float32), width, height, channels, dtype }
} finally {
inf.close();
calib.close();
}

All facades also implement [Symbol.dispose](), so under TypeScript 5.2+ / Node 20+ you can use using instead of a manual try/finally:

using calib = DepzCalibration.fromFile("calibration_stereo.json");
using inf = DepzInference.create({ arch: "lightstereo", model, engine: "openvino-CPU" });
// both closed automatically at scope exit

Filter Tuning

pipe.setFilterParam(slug, key, value) adjusts one parameter of one built-in filter without restarting the pipeline — the worker reads a fresh snapshot per frame. pipe.resetFilters() restores defaults. Every filter accepts the bool key enabled; the other keys are below. Unknown slug/key pairs throw. Float values are clamped to the ranges shown.

Slug Key Type Range / note
speckle max_size int speckle window size
speckle max_diff float disparity diff threshold
zclip min float near clip (m)
zclip max float far clip (m)
edge rel_thresh float relative edge threshold
temporal_ema alpha float EMA smoothing factor
temporal_kalman process_noise float Kalman Q
temporal_kalman measurement_noise float Kalman R
temporal_kalman adaptation_strength float adaptive gain
temporal_kalman invalid_depth_threshold float drop depth below this
bbox x_min / x_max float −20…20 (m)
bbox y_min / y_max float −20…20 (m)
bbox z_min float 0.05…20 (m)
bbox z_max float 0.1…20 (m)
stat_outlier min_neighbors int min point neighbors
stat_outlier std_ratio float 0.1…5.0
pipe.setFilterParam("temporal_kalman", "enabled", true);
pipe.setFilterParam("temporal_kalman", "measurement_noise", 0.02);
pipe.setFilterParam("zclip", "min", 0.3);
pipe.setFilterParam("zclip", "max", 6.0);

MKV Replay

Build a pipeline over a recorded .mkv instead of a live camera by passing mkvPath. A replay pipeline is seekable and supports frame-stepping; a live camera silently ignores these controls (isPausable() reports which one you have).

const pipe = DepzPipeline.build({
mkvPath: "capture.mkv",
calibrationFile: "resources/config/camera/calibration_stereo.json",
model: "resources/models/lightstereo_s_sceneflow_general_opt_256_512.onnx",
engine: "openvino-CPU",
disparity: true,
});
pipe.start();
pipe.pause();
pipe.seek(2_000_000n); // microseconds (bigint or number)
pipe.step(+1); // advance one frame while paused
const frame = pipe.get(5.0);
pipe.resume();
Method Effect
isPausable() / isPaused() whether the source supports pause, and current state
pause() / resume() hold / release the worker loop
seek(ptsUs) jump to a timestamp (µs)
step(direction) one-shot single frame while paused (+1 forward)

Notes

  • Use close() on DepzPipeline, DepzFrame, DepzSource, DepzRecorder, DepzCalibration, and DepzInference when done. Do not rely on garbage collection for native resources.
  • frame.image(stream) returns a borrowed native view. Consume it immediately.
  • frame.copyImage(stream) returns an owned Buffer.
  • Raw video, full depth maps, and point clouds should be moved as files, buffers, or browser streams. MCP and control APIs should return small JSON payloads or artifact paths.

Troubleshooting

Symptom Check
Native Node addon is not built Run npm run native:build from bindings/node, or install the packaged tarball
libdepz_istereolab.so not found Set DEPZ_LIBRARY=/abs/path/libdepz_istereolab.so, or use the self-contained tarball
pipeline_start failed on /dev/video0 Check ls -l /dev/video* and fuser -v /dev/video0; run outside sandboxes that hide camera devices
No frames within timeout Verify the camera mode, calibration file, and GStreamer pipeline can start
TypeScript cannot find Buffer or node:fs Install dependencies in bindings/node and run the type-check command shown above
Browser demo opens but image is blank Confirm the server printed a frame/FPS line and that the requested stream is enabled
