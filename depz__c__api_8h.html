Detailed Description

Flat C ABI — the language-agnostic boundary of the depz stereo-depth SDK.

Plain-C wrapper over the C++ DepthPipeline + FramePacket, exposed through opaque handles, integer error codes (no exceptions cross the FFI), and plain types (no STL in any signature). It is the single source of truth for every non-nanobind language binding — the Java Panama FFM binding and the ctypes Python binding both factor through these functions. Versioned via DEPZ_C_API_VERSION_MAJOR / DEPZ_C_API_VERSION_MINOR; verify the loaded library at runtime with depz_get_version.

Note
Threading — a depz_pipeline runs one internal worker thread. All functions on a single handle must be called from one thread at a time (not internally synchronised), EXCEPT depz_pipeline_set_* and depz_pipeline_get, which are safe to call concurrently with the worker. depz_frame handles are independent and thread-safe to read.
Lifetime — every depz_*_create returns a handle the caller must release with the matching depz_*_destroy. depz_pipeline_get returns a depz_frame the caller must release with depz_frame_destroy.
ABI stability — this header is the contract. Add new functions at the end; never reorder/remove enum values or change struct layout.
