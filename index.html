DEPZ SDK — one core, five languages

The DEPZ stereo-depth camera SDK is one C++ core exposed through a flat C ABI (depz_c_api.h): plain C, opaque handles, status codes — no C++, no exceptions. Every language binding wraps the same operations, so the architecture below reads identically whether you work in C, C++, Python, Node.js, or Java. Only the syntax changes.

Reading tip. The architecture, the pipeline, and the concepts on this page are the same for everyone. When you are ready for the API itself, jump to the one section for your language — see “Which language are you reading?” below. Each language has its own reference page and its own runnable ladder, but they all describe the same SDK.

Get started — your first depth frame in 3 steps

No matter which language you write in, getting started is the same three steps. If you can copy and paste, you can run the SDK.

Step 1 — build the core once. From a source checkout:

export ISTEREOLAB_LIBS_DIR=$HOME/libs # where the prebuilt libraries live
./sdk_master.sh build # compiles the C++ core + bindings

From a release archive instead, just run ./install.sh. (Full build options, distribution and troubleshooting live in the repository README on GitHub — this site is the API reference.)

Step 2 — pick your language and run example 01. It prints the SDK version. This is your hello world: if a version number appears, everything works.

Step 3 — climb the ladder. Change 01 to 02, then 03, … up to 15. Each step adds exactly one new idea: find a camera → show the picture → measure depth → build a 3D cloud. The same fifteen steps in every language — that is the whole SDK, one rung at a time.

The pipeline

A frame flows from the camera (or a recorded clip) to a metric depth map and a 3D point cloud. This is the same in every language — the stages live in the C++ core; your binding only starts the pipeline and reads the result.

flowchart LR SRC["camera / .mkv"] --> DEC["decode + split"] DEC --> RECT["rectify (calibration)"] RECT --> INF["inference (model)"] INF --> DISP["disparity"] DISP --> FILT["filters temporal · speckle · edge · …"] FILT --> DEPTH["depth Z = f·B / d"] DEPTH --> CLOUD["point cloud X, Y, Z"] classDef filt fill:#bf8700,stroke:#5c4000,color:#fff; class FILT filt;

Concepts (shared by all five languages)

  • Handles are objects. The core exposes five handles — source, calibration, inference, pipeline, frame. In C you pass the handle as the first argument (depz_pipeline_start(pipe)); every other language makes it the receiver (pipe.start()). Same object, same lifecycle.
  • Errors are status codes. There are no exceptions in the C ABI: a function returns DEPZ_OK (0) or a depz_status, and depz_last_error() gives a human-readable reason. The object bindings turn a failing status into their native error (a thrown exception).
  • Streams are the outputs. A frame carries the streams you asked for: raw / rectified images, disparity, depth (metres), and the point cloud (X, Y, Z).

Which language are you reading?

Pick one. Each is a full presentation of the same SDK — its own reference page, its own 15-step ladder under examples/<lang>/, identical in meaning.

  • 🟦 C — the flat C ABI, the contract everything else wraps. Reference: depz_c_api.h — every function and type on one page. Ladder: examples/c/. In C the handle is the first argument: depz_pipeline_start(pipe).
  • 🔷 C++ — depz.hpp, a header-only RAII wrapper over the C ABI. Reference: namespace depz (classes Pipeline, Calibration, Inference, Frame, Source, Recorder). Ladder: examples/cpp_cabi/. In C++ the handle is the receiver: pipe.start(). C++ alone has a second ladder, examples/cpp/, that uses the native core classes directly — the layer the C ABI wraps.
  • 🐍 Python — depz_cabi, ctypes over the C ABI. Reference: module depz_cabi (classes DepzPipeline, DepzCalibration, DepzInference, DepzFrame, DepzSource, DepzRecorder). Ladder: examples/python/. In Python the handle is the receiver: pipe.start().
  • 🟩 Node.js — N-API over the C ABI. Reference: namespace depz (classes DepzPipeline, DepzCalibration, DepzInference, DepzFrame, DepzSource, DepzRecorder). Ladder: examples/node/. Same as Python, with options objects: pipe.start().
  • ☕ Java — Panama FFM (JDK 22+), zero JNI, over the C ABI. Reference: package com.istereolab.depz (AutoCloseable wrappers DepzPipeline, DepzCalibration, …). Ladder: examples/java/ (needs JDK 22+). In Java the handle is the receiver: pipe.start().

The ladder (every language)

A 15-step learning ladder, simplest → complex, the same program in each language:

version → detect → render → rectify → disparity → depth → cloud → temporal filter → 3D → record → replay → inspect → runtime control → standalone rectify → advanced

Available under examples/c/, examples/cpp_cabi/, examples/python/, examples/node/, and examples/java/ (Java needs JDK 22+). C++ additionally has examples/cpp/, the same ladder over the native core classes. Read your language’s README there for build & run.

Versioning

The ABI is versioned via DEPZ_C_API_VERSION_MAJOR / DEPZ_C_API_VERSION_MINOR. Verify the runtime with depz_get_version(); the major number bumps on any breaking change, and every binding reports the same numbers (they all load the same core).
