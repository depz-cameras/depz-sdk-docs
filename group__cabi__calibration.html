Build the undistort/rectify remap maps for a canvas_w × canvas_h output.

Required before depz_calibration_rectify when the calibration is used standalone (outside a DepthPipeline, which otherwise calls prepare() internally). Pass the calibration's native eye size (from depz_calibration_get_info) for a 1:1 standalone rectify. Safe to call again with a different canvas size — the maps are rebuilt. (C ABI v2.2+)

Returns
DEPZ_OK on success; nonzero depz_status on error.

)

Undistort + stereo-rectify a frame pair at native resolution.

Exposes the C++ core's rectify_pair as a standalone primitive so a consumer can run capture, rectification and inference independently of the bundled DepthPipeline. Call depz_calibration_prepare first. left/right are tightly packed 8-bit images; channels is 1 (gray) or 3 (BGR, OpenCV order); the right image may be NULL for mono. Rectified output keeps the channel count. Each output buffer is written only when it fits (out_*_bytes >= *out_needed_bytes_each); out_width, out_height, out_needed_bytes_each are always set. Probe sizes by passing out_left=out_right=NULL with byte counts 0. (C ABI v1.11+)

Returns
DEPZ_OK on success; DEPZ_ERR_CALIBRATION when no maps are prepared; DEPZ_ERR_INVALID_ARG when a non-NULL buffer is too small.
