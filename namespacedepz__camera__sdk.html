Detailed Description

DEPZ Stereo SDK — public API.

This installation is the **C++/nanobind drop-in** built from the
istereolab-sdk core. Public surface is identical to the upstream
pure-python package; hot paths (sources, engines, filters, calibration,
pipeline queue, V4L2 camera enumeration) run through the native
``_core.abi3.so`` extension.

Use ``depz_camera_sdk.provenance()`` to inspect what's actually loaded
(package path, _core .so size, available engines). A one-line banner
prints on first import unless you set ``DEPZ_QUIET=1``.

Discovery helpers:

    list_connected_depz_cameras() → list[DepzCameraInfo]
    get_first_connected_depz_camera() → DepzCameraInfo
    get_connected_depz_camera_by_serial(s) → DepzCameraInfo | None
    available_engines() → list[str]
    recommend_engine()  → str | None  (respects DEPZ_ENGINE env)

dict depz_camera_sdk.provenance ( ) 
Return a dict describing what's actually loaded.

Use this to be sure the C++ drop-in is in place and not the upstream
pure-python package. Key fields:

- ``cpp_drop_in``       — True on this wheel; absent/False on upstream.
- ``package_path``      — site-packages location of `depz_camera_sdk/`.
- ``core_so``           — path to the loaded `_core` extension.
- ``core_size_kb``      — its on-disk size (sanity check).
- ``available_engines`` — engine slugs visible to this build.
- ``recommended_engine``— what recommend_engine() picks right now.
- ``camera_count``      — DEPZ cameras the platform adapter sees.
