Detailed Description

High-level camera-to-point-cloud pipeline with a built-in worker thread.

Runs the full chain — capture, rectify, disparity inference, depth, and point cloud — on a background thread and delivers each frame as a StereoPacket through a drop-oldest queue. Construct with auto-detection or inject custom components via the fluent with* builders before start().

Thread-safety: get() is the only method safe to call concurrently with the worker; configuration (with*, enable*, loadCalibration*) must precede start().

See also
DepthPipeline — the lower-level depz_camera_sdk worker loop.

Simple usage:

StereoPipeline pipe("calib.json", "", "", "openvino");
pipe.start();
auto pkt = pipe.get(1.0); // timeout in seconds
// use pkt.left.toMat(), pkt.right.toMat(), pkt.disparity.toMat()
pipe.stop();

.withInference(custom_backend)
.withFilter(custom_filter);
pipe.start();

Parameters
cameraOptional camera selector. Accepted values: "camera:N", camera ID, unique camera name, or an OS device path. Empty string → auto-select the first available camera.
calibrationPath to stereo calibration JSON file, or empty string. Empty string -> use ISTEREOLAB_CALIBRATION_FILE when set, otherwise try web calibration by resolved camera ID, then fall back to bundled calibration_stereo.json. Python bindings also accept a dict (mrcal JSON schema).
model_pathPath to inference model (.onnx/.xml/.engine). Empty string → auto-find via resources/ folder.
backendInference backend: "openvino" | "tensorrt" | "onnxruntime".
fpsTarget capture frame rate (frames per second).
max_queueMaximum number of packets buffered before drop-oldest.
inference_deviceOptional inference target override, e.g. "CPU" or "GPU". Empty string → backend default (CPU for auto high-level flow).
enable_depth_outputPopulate StereoPacket::depth.
