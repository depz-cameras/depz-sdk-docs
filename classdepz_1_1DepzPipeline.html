Detailed Description

The running worker: camera/file → rectify → inference → filters → output streams.

Build it with build, start, pull frames with get, then close.

One-call raw camera pipeline — no calibration, no model (the C depz_pipeline_create_camera).

The device may be a path, an ID/serial, a name, or an index; the pipeline builds and owns the camera. Mirrors Python/Java DepzPipeline.camera(...) — a raw camera needs nothing but itself.

◆ setFilterParam()

depz::DepzPipeline::setFilterParam (   slug,
  key,
  value 
)
inline

Tune one filter live, without restarting the pipeline.

For example the temporal EMA filter smooths each pixel over time:

\[ \mathrm{out}_t = \alpha\,\mathrm{new}_t + (1-\alpha)\,\mathrm{out}_{t-1} \]

so setFilterParam("temporal_ema", "alpha", 0.2) sets \(\alpha\).

Parameters
slugFilter name (e.g. "temporal_ema", "speckle").
keyParameter name within that filter.
valuenumber or boolean — routed to the matching C setter by type.
