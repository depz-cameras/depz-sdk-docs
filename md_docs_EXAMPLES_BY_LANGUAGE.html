4. Tune a filter control (live, no restart)

The slug/key set is shared by every binding (it is C-ABI level): filters speckle · zclip · edge · temporal_ema · temporal_kalman · bbox · stat_outlier, every one accepts the boolean enabled.

6. MKV replay — seek & step

Build over a .mkv instead of a camera; a replay pipeline is seekable and frame-steppable (a live camera ignores these — check isPausable / is_pausable).

Java

try (DepzPipeline pipe = DepzPipeline.builder()
.mkvSource(Path.of("capture.mkv"))
.calibrationFromFile(Path.of("calibration_stereo.json"))
.inference("lightstereo", modelPath, null).enableDisparity(true)
.build()) {
pipe.start();
pipe.pause();
pipe.seek(2_000_000L); // µs
pipe.step(+1);
pipe.resume();
}

Python

pipe = DepthPipeline.build("capture.mkv",
inference=("lightstereo", "s-sceneflow-opt-256x512"),
enable_disparity=True)
pipe.start()
pipe.pause()
pipe.seek(2_000_000) # µs
pipe.step_replay(+1) # one frame forward while paused
pipe.resume()

Node / TypeScript

const pipe = DepzPipeline.build({ mkvPath: "capture.mkv",
calibrationFile: "calibration_stereo.json", model, disparity: true });
pipe.start();
pipe.pause();
pipe.seek(2_000_000n); // µs (bigint or number)
pipe.step(+1);
pipe.resume();

seek and metrics are the canonical names in every binding (the C ABI function names depz_pipeline_seek / depz_pipeline_get_metrics, case-folded).

Resource cleanup — the one thing that is the language's, not ours

Every binding frees the same destroy morphism, idiomatically:

Language Idiom
C explicit depz_*_destroy(...)
C++ destructor (RAII)
Java try-with-resources / close()
Python with / close()
Node / TS using + [Symbol.dispose] / close()

close() exists in every OOP binding as the common form.
