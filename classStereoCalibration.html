Detailed Description

Stereo calibration: load from JSON, rectify images, expose matrices.

Thread-safety: the rectify* methods are read-only and safe to call concurrently after loadFromFile() or loadFromJsonString() has returned. Mutating methods (setTargetSize, setValue) are not thread-safe.

See also
CameraCalibrationWebLoader — to download calibration over HTTP by camera serial.
IStereoCalibrationModel — the pure-virtual interface this class satisfies.

◆ getQMatrix()

cv::Mat StereoCalibration::getQMatrix ( ) const
inlineoverridevirtual

Disparity-to-3D reprojection matrix \(Q\) (4×4, CV_64F).

Python binding:
istereolab.get_q_matrix

Maps a pixel \((u,v)\) with disparity \(d\) to a homogeneous 3-D point, which is then de-homogenised:

\[ \begin{bmatrix} X \\ Y \\ Z \\ W \end{bmatrix} = Q\, \begin{bmatrix} u \\ v \\ d \\ 1 \end{bmatrix}, \qquad (X, Y, Z) \;\leftarrow\; (X/W,\; Y/W,\; Z/W). \]

For a rectified pair the depth term reduces to \(Z = f B / d\), so the 3-D coordinates inherit the unit of the baseline \(B\) (millimetres here). Consumed by cv::reprojectImageTo3D and depthToPointCloud().

Implements stereocam::IStereoCalibrationModel.

Rectify both images of a stereo pair so epipolar lines become horizontal rows.

Python binding:
istereolab.rectify_pair

Each camera carries the pinhole intrinsics \(K\) and the OpenCV rational distortion vector \((k_1,k_2,p_1,p_2,k_3,\dots)\). A normalized point \((x,y)\), \(r^2 = x^2 + y^2\), distorts to

\[ x_d = x\,(1 + k_1 r^2 + k_2 r^4 + k_3 r^6) + 2 p_1 x y + p_2 (r^2 + 2x^2), \]

\[ y_d = y\,(1 + k_1 r^2 + k_2 r^4 + k_3 r^6) + p_1 (r^2 + 2y^2) + 2 p_2 x y, \]

with the pixel projection \(u = f_x x_d + c_x,\; v = f_y y_d + c_y\).

Rectification applies the per-camera rotations \(R_1, R_2\) that share a common image plane, so the rectified projection matrices

\[ P_1 = \begin{bmatrix} f & 0 & c_x & 0 \\ 0 & f & c_y & 0 \\ 0 & 0 & 1 & 0 \end{bmatrix}, \quad P_2 = \begin{bmatrix} f & 0 & c_x & -f B \\ 0 & f & c_y & 0 \\ 0 & 0 & 1 & 0 \end{bmatrix} \]

share focal length \(f\) and principal row \(c_y\), differing only by the baseline term \(-f B\). The two undistort-and-rectify maps are resampled with cv::remap; after this call corresponding features lie on the same image row and disparity reduces to a horizontal search.

Parameters
[in]leftRaw (distorted) left image.
[in]rightRaw (distorted) right image.
[out]leftRectRectified left image, epipolar-aligned with rightRect.
[out]rightRectRectified right image.
Returns
true on success.
