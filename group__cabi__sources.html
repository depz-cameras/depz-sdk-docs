DEPZ_API depz_status depz_source_create_pipeline ( const char *  gst_pipeline,
depz_source **  out 
)

Create a GStreamer source from an arbitrary gst-launch pipeline string.

gst_pipeline is any gst-launch description, e.g. "v4l2src device=/dev/video0 ! image/jpeg ! jpegdec ! videoconvert ! appsink name=appsink" "videotestsrc ! videoconvert" "rtspsrc location=... ! decodebin ! videoconvert". If the string contains no appsink, a BGR-converting appsink name=appsink is appended automatically; a caller-supplied appsink is used as-is. (C ABI v2.1+)

Parameters
[in]gst_pipelinegst-launch pipeline description string.
[out]outReceives the new depz_source handle (NULL on failure; free via depz_source_destroy).
Returns
DEPZ_OK on success; nonzero depz_status on error.

DEPZ_API depz_status depz_source_grab ( depz_source *  src,
double  timeout_s,
uint8_t *  out_buf,
size_t  buf_size,
size_t *  out_size,
int64_t *  out_pts_us 
)

Grab the next frame's raw encoded payload, blocking up to timeout_s.

For a DEPZ camera the payload is one side-by-side MJPG image (JPEG for fourcc "MJPG" — decode with any JPEG decoder). *out_size always receives the true payload size and *out_pts_us its timestamp (microseconds). When the frame is larger than buf_size nothing is copied and DEPZ_ERR_INVALID_ARG is returned (grow the buffer and grab again); pass out_buf=NULL, buf_size=0 to probe a typical size first.

Returns
DEPZ_OK on success; DEPZ_ERR_TIMEOUT when no frame arrives in the window; DEPZ_ERR_INVALID_ARG when the buffer is too small.
