Detailed Description

Per-pipeline recording handle.

Constructed once at pipeline init; multiple `start`/`stop` cycles
are supported on the same controller (each `start` builds a fresh
MkvRecorder). When stopped, holds no recorder reference, so
`is_running` returns False.

``info_provider`` is a thunk pointing at the pipeline's current
:class:`CameraInfo` (set by ``pipeline.start()``); evaluated at
:meth:`start` time. Callable rather than a direct field so the
controller doesn't grow a back-reference into pipeline internals.

Live-mux one online-recal entry into the recording's
``depz_metadata`` subtitle stream. ``pts_us`` is source-local
(matches :attr:`RawFrame.pts_us` the worker is currently at);
the recorder converts to recording-relative before muxing.
No-op when not recording. Designed for future periodic-recal
subsystems that want to checkpoint state at each successful
patch.
