Detailed Description

Geometry of the rectified eye inside the model's fixed-size canvas.

Produced by Calibration::compute_canvas_remap (and its Python counterpart). The neural model sees a canvas_w × canvas_h image, but only valid_w × valid_h of it carries real rectified content; the remainder is letterbox padding in fit-with-padding mode, or off-canvas content the rectifier did not draw in fill-with-cropping mode. Postprocess slices the disparity/depth output back to the (pad_x, pad_y)-anchored valid region so downstream stages never see the padding.

This is the SDK-core counterpart of the Python @dataclass CanvasLayout in depz_camera_sdk.calibration.base; the binding layer exposes it unchanged to keep the Python type identity.
