iStereoLab SDK  0.0.10
C++17 stereo vision library with Python bindings
Java Binding

The Java SDK is an idiomatic wrapper over the depz flat C ABI (include/depz/c_api/depz_c_api.h), published from bindings/java as com.istereolab:depz-sdk. It is zero JNI / zero C++ glue — pure Java over the Panama Foreign Function & Memory API.

Runtime path:

Java
-> com.istereolab.depz (MethodHandle per depz_* C function)
-> Panama FFM (SymbolLookup.libraryLookup)
-> flat C ABI
-> C++ core

The same core, same C ABI, and same 69 functions as the ctypes-Python and Node bindings — so the Java numbers match those bindings byte-for-byte (see docs/CROSS_LANGUAGE_INVARIANTS.md).

Requirements

  • Linux x86_64
  • JDK 22 or newer — the Panama FFM API is stable from JDK 22; run with --enable-native-access=ALL-UNNAMED
  • DEPZ native runtime libraries: bundled inside the jar / java-dist tarball, or available from a local SDK build
  • A stereo camera such as /dev/video0, or an MKV replay file for pipeline examples that need frames

End User Quickstart

Depend on the jar (native core bundled inside, extracted + loaded at runtime):

dependencies { implementation("com.istereolab:depz-sdk:0.0.5") }

A consumer needs nothing on the system but a JDK 22+. Run with native access enabled:

java --enable-native-access=ALL-UNNAMED -cp depz-sdk-0.0.5.jar:app.jar MyApp

Run an example from this repository:

cd examples/java
./run.sh HelloDepz # default; compiles every *.java then runs the class
./run.sh HelloStereoImages

run.sh finds a JDK 22+ under ~/libs/jdk (then PATH), loads libdepz_istereolab.so from <repo>/build, and uses a sample calibration from <repo>/resources.

Build From Source

A Gradle wrapper is committed — no system Gradle needed; ./gradlew fetches its own toolchain. Build the C++ core first.

export ISTEREOLAB_LIBS_DIR=$HOME/libs
./sdk_master.sh build # → build/libdepz_istereolab.so
cd bindings/java
./gradlew build # compile + bundle → build/libs/depz-sdk-<ver>.jar
./gradlew run # run Demo (needs a camera + a cached model)

gradle run points the loader at the in-repo core via -Ddepz.library.path.

Without Gradle, compile and run directly:

JDK=$(ls ~/libs/jdk/jdk-*/bin/javac | head -1); J=${JDK%javac}java
"$JDK" --release 23 -d /tmp/out src/main/java/com/istereolab/depz/*.java
"$J" --enable-native-access=ALL-UNNAMED \
-Ddepz.library.path="$PWD/../../build/libdepz_istereolab.so" \
-cp /tmp/out com.istereolab.depz.Demo

Package Artifact

Build the self-contained Java tarball (jar + bundled native stack + launcher):

export ISTEREOLAB_LIBS_DIR=$HOME/libs
./sdk_master.sh cpp-dist # build first so java-dist reuses its richer lib/
./sdk_master.sh java-dist # → dist/depz-sdk-java-<ver>-linux-x86_64.tar.gz

The artifact is written under dist/ as:

depz-sdk-java-<version>-linux-x86_64.tar.gz

It contains the thin depz-sdk-<ver>.jar, a lib/ with the native core and every non-system dependency (OpenCV / OpenVINO / ONNX, RPATH=$ORIGIN), the compiled examples, sample resources, and a run.sh launcher. Running cpp-dist first lets java-dist reuse the C++ tarball's lib/, which also bundles the GStreamer plugins needed for the live camera.

Java Use

DepzPipeline is AutoCloseable with a fluent builder; use it in try-with-resources so close() tears down the pipeline (and the source / calibration / inference it owns).

import com.istereolab.depz.*;
import java.nio.file.Path;
try (DepzPipeline pipe = DepzPipeline.builder()
.camera("/dev/video0", 1280, 480, 20)
.calibrationFromFile(Path.of("calibration_stereo.json")) // or .calibrationFromJson(str) / .calibrationFromWeb(url)
.inference("lightstereo", modelPath, null) // engine null → recommended
.enableRectified(true).enableDisparity(true).enableDepth(false)
.build()) {
pipe.start();
try (DepzFrame f = pipe.get(5.0)) { // null on timeout
if (f != null) {
DepzImage disp = f.image(Stream.DISPARITY); // F32, copied out — safe after close
float px = disp.floatAt(disp.width() / 2, disp.height() / 2);
float metres = f.sampleDepth(disp.width() / 2, disp.height() / 2);
}
}
pipe.stop();
} // close() = stop + destroy pipeline → inference → calibration → source

Standalone Classes

DepzPipeline is the all-in-one path. When you need finer control — your own capture loop, rectification without inference, direct disparity, or recording — drive the building blocks directly. Each is created through a static factory and is AutoCloseable (close() / try-with-resources).

Class Factories Key methods
DepzSource .camera(device,w,h,fps), .mkv(path), .pipeline(gst) open(), grab(timeoutS), close(), closeDevice()
DepzCalibration .fromFile(path), .fromJson(json), .fromWeb(url) info(), setCanvasFitMode(mode), prepare(w,h), canvasLayout(), rectify(...)
DepzInference .create(arch, model, engine) setIntrinsics(fx,fy,cx,cy), disparity(left,right) → DepzImage
DepzRecorder .create(path,w,h,fps,fourcc) write(data, ptsUs), close()
try (DepzCalibration calib = DepzCalibration.fromFile(Path.of("calibration_stereo.json"));
DepzInference inf = DepzInference.create("lightstereo", modelPath, "openvino-CPU")) {
calib.prepare(width, height);
// left / right: raw BGR byte[] from your own capture + split-at-mid-column.
DepzCalibration.Rectified rect =
calib.rectify(left, width, height, right, width, height, 3);
DepzImage disp = inf.disparity(
rect.left(), rect.width(), rect.height(),
rect.right(), rect.width(), rect.height(), 3); // → F32 disparity
}

Filter Tuning

pipe.setFilterParam(slug, key, value) adjusts one parameter of one built-in filter without restarting; pipe.resetFilters() restores defaults. Every filter accepts the boolean key enabled. The slug/key set is shared with every binding (it is C-ABI level) — see the table in docs/NODE_JS_TYPESCRIPT.md:

Slug Keys
speckle max_size (int), max_diff (float)
zclip min, max (float, metres)
edge rel_thresh (float)
temporal_ema alpha (float)
temporal_kalman process_noise, measurement_noise, adaptation_strength, invalid_depth_threshold (float)
bbox x_min/x_max/y_min/y_max (−20…20), z_min (0.05…20), z_max (0.1…20) (float, m)
stat_outlier min_neighbors (int), std_ratio (0.1…5.0)
pipe.setFilterParam("temporal_kalman", "enabled", true);
pipe.setFilterParam("zclip", "min", 0.3f);
pipe.setFilterParam("zclip", "max", 6.0f);

MKV Replay

Build a pipeline over a recorded .mkv with .mkvSource(path). A replay pipeline is seekable and supports frame-stepping; a live camera ignores these controls (isPausable() reports which one you have).

try (DepzPipeline pipe = DepzPipeline.builder()
.mkvSource(Path.of("capture.mkv"))
.calibrationFromFile(Path.of("calibration_stereo.json"))
.inference("lightstereo", modelPath, null)
.enableDisparity(true)
.build()) {
pipe.start();
pipe.pause();
pipe.seek(2_000_000L); // microseconds
pipe.step(+1); // one frame forward while paused
try (DepzFrame f = pipe.get(5.0)) { /* ... */ }
pipe.resume();
}
Method Effect
isPausable() / isPaused() whether the source supports pause, and current state
pause() / resume() hold / release the worker loop
seek(ptsUs) jump to a timestamp (µs)
step(direction) one-shot single frame while paused (+1 forward)

Native Loading — resolution order

  1. -Ddepz.library.path=/abs/path/lib… (or DEPZ_LIBRARY env) — load it directly. Best for development against build/.
  2. Bundled in the jar at /native/<os>-<arch>/<libname> — extracted to temp and loaded.
  3. <cwd>/build/<libname> — convenience fallback when run from the repo root.

Notes

  • Use try-with-resources (or explicit close()) on DepzPipeline, DepzFrame, DepzSource, DepzRecorder, DepzCalibration, and DepzInference. Do not rely on the GC for native resources.
  • DepzFrame.image(stream) returns a copied-out DepzImage that stays valid after the frame closes (unlike the Node borrowed view).
  • A non-DEPZ_OK C status becomes a DepzException (unchecked); a frame timeout is a null from get(), not an exception.
  • Raw video, full depth maps, and point clouds should move as files, buffers, or browser streams — keep control APIs returning small payloads.
  • Wraps 100% of the C ABI (69/69, verified by tests/conformance/check_abi.py).

Troubleshooting

Symptom Check
UnsatisfiedLinkError / library not found Set -Ddepz.library.path= or DEPZ_LIBRARY= to the .so, or use the java-dist tarball
IllegalCallerException / native access denied Add --enable-native-access=ALL-UNNAMED
Needs JDK 22+ Panama FFM is stable from JDK 22; older JDKs cannot run this binding
pipeline start failed on /dev/video0 Check ls -l /dev/video* and fuser -v /dev/video0; run outside sandboxes that hide camera devices
No frames within timeout Verify the camera mode, calibration file, and that GStreamer can start
Live camera fails from the tarball Run sdk_master.sh cpp-dist before java-dist so the bundled lib/ carries the GStreamer plugins
