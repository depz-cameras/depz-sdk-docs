DEPZ SDK — Flat C ABI (depz_c_api.h)

Source of truth: include/depz/c_api/depz_c_api.h · impl src/c_api/depz_c_api.cpp · current version 2.3 (DEPZ_C_API_VERSION_MAJOR.MINOR) · 69 exported functions.

The flat C ABI is the facade of the SDK — the single stable external interface that every consumer programs against. C++ has no stable ABI, so the C++ core (DepthPipeline, FramePacket, calibration, inference, filters) is never touched directly from outside; it is reached only through the ~69 functions in this header. The Python (nanobind), Java (Panama FFM), and ctypes bindings, plus any C / C++ / embedded consumer, all enter the SDK through exactly these functions. The facade wraps the core behind opaque handles, integer error codes (no exceptions cross the FFI), and plain types (no STL in any signature).

Design rules baked into the header:

  • Add new functions at the end; never reorder/remove enum values or change a struct layout. Additive change → minor bump; breaking rename/layout → major bump.
  • Every depz_*_create returns a handle the caller releases with the matching depz_*_destroy. depz_pipeline_get returns a depz_frame to release with depz_frame_destroy.
  • One worker thread per pipeline. All calls on a handle are single-threaded except depz_pipeline_set_* and depz_pipeline_get, which are safe to call concurrently with the worker.

The two C-ABI bindings (Java FFM and ctypes-Python) factor through the same C functions, so their numbers agree byte-for-byte. nanobind is a separate surface over the C++ classes and does not track C-ABI changes automatically (see the cross-language category notes in the SDK repo).

Binding-sync rule: a C-ABI change touches four files — depz_c_api.h + depz_c_api.cpp, plus both C-ABI bindings (bindings/java/.../DepzNative.java, bindings/python_cabi/depz_cabi.py) — then tests/conformance/check_abi.py must stay green.

Reference — every function by subsystem

Each of the ~69 facade functions, grouped into the same subsystems as the Doxygen Modules tree. Skim by ### <Subsystem>; each #### \name`` carries the signature and one purpose sentence.

Return convention — every function that returns depz_status yields DEPZ_OK (0) on success and a nonzero depz_status on error; call depz_last_error() for human-readable text for the last failure on the calling thread. Functions returning void, int32_t, int64_t, or double cannot fail this way and are noted as such. Per-function return notes below stay terse — only deviations from this convention (specific error codes, timeout, too-small buffer) are called out.

Ownership — every depz_*_create* (and depz_pipeline_get) hands back a handle the caller must release with the matching depz_*_destroy; the owning _destroy is named in each _create* entry. Any source / calibration / inference handle passed to depz_pipeline_create is referenced, not copied — it must outlive the pipeline; destroy the pipeline first, then its dependencies.

Worked example — AI disparity in C

#include <depz/c_api/depz_c_api.h>
depz_source* src; depz_calibration* cal; depz_inference* inf; depz_pipeline* pipe;
depz_source_create_camera("/dev/video0", 1280, 480, 30, "MJPG", &src);
depz_calibration_create_from_web(
"https://<host>/functions/v1/camera-calibration?serial_number=RRRAPDXIFD", &cal);
/* lightstereo on NVIDIA TensorRT — resolves the sibling model_fp16.engine */
depz_inference_create("lightstereo", "/cache/.../model.onnx", "tensorrt-cuda", &inf);
depz_pipeline_config cfg;
depz_pipeline_config_default(&cfg);
cfg.enable_disparity = 1;
depz_pipeline_create(src, cal, inf, &cfg, &pipe);
depz_pipeline_start(pipe);
depz_frame* frame;
if (depz_pipeline_get(pipe, 2.0, &frame) == DEPZ_OK) {
size_t need = 0;
depz_frame_encode_jpeg(frame, DEPZ_STREAM_DISPARITY, -1, 90, NULL, 0, &need);
uint8_t* jpg = malloc(need);
depz_frame_encode_jpeg(frame, DEPZ_STREAM_DISPARITY, -1, 90, jpg, need, &need);
/* jpg/need is a coloured disparity JPEG — write it, stream it, display it */
free(jpg);
depz_frame_destroy(frame);
}
depz_pipeline_stop(pipe);
depz_pipeline_destroy(pipe);
depz_inference_destroy(inf);
depz_calibration_destroy(cal);
depz_source_destroy(src);

Engine slugs

depz_inference_create(arch, model_path, engine_slug, …) — engine_slug is "{kind}-{device}": openvino-CPU, openvino-GPU, onnx-cpu, onnx-cuda, **tensorrt-cuda** (native TensorRT; resolves the sibling model_fp16.engine). depz_list_engines / depz_recommend_engine report what the running machine supports, highest-priority first.
